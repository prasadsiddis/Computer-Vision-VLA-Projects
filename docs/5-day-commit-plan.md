# Five-Day Commit Plan

Use this plan to create small, real updates over the next few days. Each item should be a focused commit with a clear message.

## Day 1: Add Notebook Output Tables

Run one or two notebooks and save example output tables under `outputs/examples/`.

Suggested commit message:

```text
Add example VLA decision traces
```

## Day 2: Add Dataset Adapter Notes

Add notes for converting one official dataset into the sample manifest schema. Start with nuScenes mini or KITTI.

Suggested commit message:

```text
Document nuScenes mini adapter plan
```

## Day 3: Add Failure-Case Examples

Create a short markdown file with two or three scenes where the baseline policy is likely to fail.

Suggested commit message:

```text
Add VLA baseline failure cases
```

## Day 4: Add Experiment Output Screenshots

Run selected notebook cells and add screenshots or exported plots from the outputs.

Suggested commit message:

```text
Add notebook output screenshots
```

## Day 5: Add Research Write-Up

Write a short summary connecting the portfolio to autonomous-driving VLA research.

Suggested commit message:

```text
Add autonomous driving VLA research summary
```

## Rule

Keep each commit small and real. Do not change dates manually. The best signal is steady, understandable project progress.
